Router and MVC

  • Routing
    • To lighten and heavy code and make it look neater, routes can be separated into a different file.
    • Express allows making a separate router file.
    • When the size of the project grows with many different routes, management becomes easier.
    • One big messy file can be avoided.
    • In this part, we make a new folder called "routes", with a new file called "blogRoutes"
    • To use router functionality like this, the file should have express and should an instance of router running.
    • At the end of the file, the routing object should be exported also.
    • The file should be imported to the app file using require function.
    • To use blogRoutes in the app, we can use the "use" funciton of the node just like using any middleware.
    • The routes can be scoped using router like this too.
  • MVC
    • MVC stands for Model, View and Controller
    • it is a way of structuring code and files
    • It keeps code modular, reusable and easier to read.
    • Model and View is already covered, controller which is the middle man betweeen the former too will be covered now.
    • Controller files contains functions to magage the tasks that involve model and views.
    • In this project, all the route handler functions are extracted into a separate controller file.
    • A separate controllers folder is created.
    • It will have all the functions for all the different type of operations that are done on the blogs.
    • This makes the app much more handleable because of modularity and future mods can be easily done.
